# Architecture Decision Records

ADRs capture technical decisions with meaningful tradeoffs.

Use `TEMPLATE.md` for new ADRs.

Create or update an ADR when a change affects:

- architecture
- data model
- storage
- permissions
- extraction or OCR flows
- tenant defaults or destructive migration behavior
- important dependencies
- decisions with non-obvious tradeoffs

Do not create ADRs for minor UI changes, simple fixes, renames, copy changes, or refactors without architectural impact.

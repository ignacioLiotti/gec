# AGENTS.md

## Cursor Cloud specific instructions

### Overview

Sintesis is a multi-tenant construction project management platform built with **Next.js 16** (App Router, Turbopack) and **Supabase** (Postgres + Auth + Storage). The codebase is a single Next.js app (not a monorepo). Deployment target is Vercel; local dev uses the Supabase CLI to run a local Postgres/Auth/Storage stack via Docker.

### Services

| Service | How to run | Port | Notes |
|---|---|---|---|
| Next.js dev server | `npm run dev` | 3000 | Main app; uses Turbopack |
| Supabase local stack | `npx supabase start` | API: 54321, DB: 54322, Studio: 54323, Mailpit: 54324 | Requires Docker; runs Postgres, GoTrue, PostgREST, Storage, etc. |

### Key commands

See `package.json` scripts. Highlights:

- **Lint:** `npm run lint`
- **Test:** `npm run test` (Vitest)
- **Dev server:** `npm run dev`
- **Supabase start/stop:** `npm run supabase:start` / `npm run supabase:stop`
- **DB reset (migrations + seed):** `npm run db:reset`

### Gotchas

- **`.npmrc` has `legacy-peer-deps=true`** — always use `npm install --legacy-peer-deps` or just `npm install` (the `.npmrc` setting handles it).
- **`env.example` anon key is stale** — the key in `env.example` does not match the one generated by `supabase start`. After copying `env.example` to `.env.local`, update `NEXT_PUBLIC_SUPABASE_ANON_KEY` with the value from `npx supabase status -o env` (look for `ANON_KEY`).
- **`SUPABASE_SERVICE_ROLE_KEY` must be set** — organization creation (onboarding) fails without it. Get it from `npx supabase status -o env` (look for `SERVICE_ROLE_KEY`) and add it to `.env.local`.
- **Docker socket permissions** — in Cloud Agent VMs, after installing Docker you may need `sudo chmod 666 /var/run/docker.sock` before `supabase start` will work.
- **Google OAuth warnings** — `supabase start` emits warnings about unset `GOOGLE_CLIENT_ID`/`GOOGLE_CLIENT_SECRET`. These are safe to ignore for local dev; email/password auth works fine without them.
- **Lint has pre-existing errors** — `npm run lint` exits non-zero due to ~400 pre-existing ESLint errors in the codebase. This is expected.
- **Workspace rule: do not run `npm run build` or `npm run dev`** when the user already has these running locally. In Cloud Agent VMs (fresh environment), running `npm run dev` is fine.

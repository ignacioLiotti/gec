## Findings considered and rejected

- **"Committed .env secrets"** (auditor claim): false — `.gitignore` covers `.env*`; `git ls-files` and full history show no `.env` ever tracked. No action.
- **CSRF via `sec-fetch-site` header**: by-design modern pattern; absent header must pass for webhooks/non-browser clients. No action.
- **MercadoPago webhook idempotency / out-of-order overwrites**: mitigated by design — the handler re-fetches preapproval state from MercadoPago's API instead of trusting payloads. No action (signature enforcement is plan 004).
- **`fetchAllMacroTableRows` "missing pagination"**: mis-reported — it paginates server-side in 200-row pages (`lib/obra-queries.ts:383-421`); sequential accumulation is a minor perf note, below the line.
- **npm-audit criticals in `basic-ftp`/`vitest`**: devDependencies, no production runtime exposure. The production-relevant advisory is `xlsx` → plan 009.
- **`@supabase/supabase-js` "outdated"**: caret range, no evidenced vulnerability. No action.
- **Request-signing 60s secret cache window**: deliberate TTL trade-off, marginal risk. No action.
- **CSP `frame-src` missing**: `frame-ancestors 'none'` already prevents framing. No action.
- **CSP `unsafe-eval` + unpkg.com in script-src** (real, finding #9): acknowledged but NOT planned this round — requires a Next.js nonce/CSP rework with broad regression surface; revisit after the P1 security plans land.
- **`PERF-01` sequential macro fetches, `PERF-04` select("\*") over-fetch, `PERF-06` thumbnail queue debounce**: real but small; below the line vs. selected plans. Candidates for a future perf pass.
- **Obra `DEFAULT_TENANT_ID` fallback for membership-less users** (`proxy.ts:321`): needs a product decision; flagged in plan 005's maintenance notes rather than planned.
